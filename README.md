## معرفی پروژه

این مخزن مجموعه ای ساختارمند از مثال ها، تمرین ها و پاسخ های آموزشی درس «برنامه نویسی موبایل 1» است که با تمرکز بر مفاهیم پایه Java و اصول برنامه نویسی شیءگرا تهیه شده است. محتوای پروژه به صورت سشن بندی شده تنظیم شده تا مسیر یادگیری از مفاهیم مقدماتی تا مباحث مهمی مانند کپسوله سازی، وراثت، پیاده سازی Interface، بازنویسی، بارگذاری متدها، اعضای ایستا و کلاس های انتزاعی به شکل مرحله ای دنبال شود.

هدف این مخزن صرفا نگهداری چند فایل کد نیست، بلکه فراهم کردن یک مرجع آموزشی قابل فهم برای دانشجویان، مدرسین و توسعه دهندگان تازه وارد است تا بتوانند هم ساختار پروژه ها را سریع درک کنند و هم ارتباط بین مفاهیم مختلف را در قالب مثال های مستقل مشاهده کنند.

## توضیح بخش های اصلی و هدف هر کدام

### 1. `Variables (session 1)`

این بخش نقطه شروع یادگیری است و بر تفاوت بین داده های Primitive و Reference تمرکز دارد.

- `Refrens AND primitive/src/Main.java`: نمایش نمونه های اولیه از انواع داده، متغیرها و اشیاء
- `Refrens vs primitive/src/Main.java`: مقایسه رفتاری داده های مرجع و داده های اولیه

**هدف آموزشی:** ایجاد درک اولیه از نحوه ذخیره و استفاده از داده ها در Java، که پایه فهم تمامی بخش های بعدی است.

### 2. `Encapsulation (session 2)`

این بخش مفهوم کپسوله سازی را با چند سناریوی مقایسه ای نمایش می دهد. برخی پروژه ها پیاده سازی درست را نشان می دهند و برخی دیگر نمونه های بدون کپسوله سازی یا حالت های خطادار را برای درک بهتر تفاوت ها ارائه می کنند.

- `BankAccount.java`: نمایش کنترل دسترسی و مدیریت داده های حساس در سناریوی حساب بانکی
- `Student.java`: نمونه ای روشن از استفاده از `private`، `getter` و `setter`
- `Car.java`: استفاده در مثال های Crash برای نمایش اثر طراحی نادرست یا نبود کنترل اعتبار
- `Main.java`: اجرای سناریو و نمایش نتیجه رفتاری هر مدل

**هدف آموزشی:** نشان دادن این که چرا پنهان سازی داده، اعتبارسنجی ورودی و کنترل دسترسی برای طراحی امن و قابل نگهداری ضروری است.

### 3. `Extends (session 3)`

این بخش برای آموزش وراثت کلاس ها طراحی شده است و نشان می دهد چگونه یک کلاس می تواند ویژگی ها و رفتارهای کلاس پایه را به ارث ببرد.

- `Vehicle.java`: کلاس پایه با ویژگی ها و رفتار عمومی
- `Car.java`: کلاس مشتق شده که از کلاس والد استفاده یا آن را توسعه می دهد
- `Bird.java`: نمونه دیگر از توسعه رفتار در ساختار آموزشی متفاوت
- `FlyAble.java`: رابط یا ساختار کمکی برای نمایش ارتباط بین توانایی پرواز و کلاس ها
- `Main.java`: اجرای سناریوی وراثت

**هدف آموزشی:** درک رابطه پدر و فرزند، استفاده مجدد از کد و مدل سازی سلسله مراتبی اشیاء.

### 4. `Implements (session 3)`

این بخش روی پیاده سازی Interface متمرکز است و چندین مثال را برای سناریوهای تک رابطی و چند رابطی ارائه می دهد.

- `Animal.java`, `Pig.java`: نمونه پیاده سازی رفتار از طریق Interface یا قرارداد رفتاری
- `FirstInterface.java`, `SecondInterface.java`: تعریف قراردادهای مستقل
- `DemoClass.java`: پیاده سازی همزمان چند Interface
- `Swimmable.java`: تعریف قابلیت شنا برای چند موجود یا شیء مختلف
- `Duck.java`, `Fish.java`, `Human.java`: نمایش استفاده مشترک از یک قرارداد رفتاری واحد
- `Main.java`: راه اندازی و تست تعامل بین کلاس ها و Interface ها

**هدف آموزشی:** آموزش قراردادهای رفتاری، جداسازی وابستگی ها و تفاوت بین وراثت کلاس و پیاده سازی Interface.

### 5. `Overloading (session 3)`

این بخش مفهوم بارگذاری متد را با یک مثال ساده و قابل فهم پیاده سازی می کند.

- `Calculator.java`: شامل متدهایی با نام یکسان و پارامترهای متفاوت
- `Main.java`: فراخوانی نسخه های مختلف متدها برای مشاهده تفاوت رفتار

**هدف آموزشی:** درک چندریختی ایستا و نحوه انتخاب متد بر اساس امضای ورودی.

### 6. `Overriding (session 3)`

این بخش به بازنویسی متد در کلاس های فرزند می پردازد.

- `Animal.java`: تعریف رفتار پایه که در کلاس های دیگر قابل بازنویسی است
- `Main.java`: اجرای رفتار بازنویسی شده و مقایسه آن با پیاده سازی پایه

**هدف آموزشی:** درک چندریختی پویا و تغییر رفتار در زمان اجرا.

### 7. `Static (session 4)`

این بخش برای نمایش نقش اعضای `static` در Java طراحی شده است.

- `Static 1/src/Main.java`
- `Static 2/src/Main.java`
- `Static 3/src/Main.java`

هر مثال بخشی از رفتار متغیرها یا متدهای ایستا را نمایش می دهد.

**هدف آموزشی:** درک تفاوت اعضای وابسته به کلاس و اعضای وابسته به شیء.

### 8. `Abstract (session 4)`

این بخش بر کلاس ها یا مفاهیم انتزاعی تمرکز دارد.

- `Abstract 1/src/Main.java`
- `Abstract 2/src/Main.java`

**هدف آموزشی:** آشنایی با انتزاع، محدود کردن پیاده سازی مستقیم و طراحی مبتنی بر تعریف رفتارهای عمومی.

### 9. `Exercises`

این پوشه شامل فایل های تمرینی و پاسخ ها است و مکمل کدهای آموزشی محسوب می شود.

- `Exercises with answer.docx`
- `Exercises with answer.pdf`
- `Practice without answers.docx`
- `Practice without answers.pdf`

**هدف آموزشی:** تمرین مستقل، ارزیابی یادگیری و مقایسه پاسخ ها با راه حل های نمونه.

### 10. فایل های سطح ریشه

- `README.md`: مستند اصلی پروژه و راهنمای آشنایی با ساختار مخزن
- `questions.docx`: فایل کمکی یا سوالات مرتبط با محتوای آموزشی

## نقش فایل ها و ماژول های کلیدی

در اغلب زیرپروژه ها، فایل های اصلی نقش های ثابتی دارند:

- `Main.java`: نقطه ورود اجرای برنامه و نمایش نحوه استفاده از کلاس ها
- فایل های مدل مانند `Student.java`، `BankAccount.java`، `Vehicle.java`، `Car.java`: پیاده سازی منطق اصلی هر مثال
- فایل های قراردادی مانند `FirstInterface.java`، `SecondInterface.java`، `Swimmable.java`, `FlyAble.java`: تعریف رفتارهایی که کلاس ها باید پیاده سازی کنند
- فایل های تنظیمات IntelliJ مانند `.idea/` و `.iml`: پیکربندی محیط توسعه و مدیریت پروژه در IDE

این الگو به توسعه دهندگان جدید کمک می کند تا بدون جست وجوی طولانی، تشخیص دهند نقطه شروع اجرا کجاست، منطق اصلی در کدام فایل ها قرار دارد و قراردادهای رفتاری در چه فایل هایی تعریف شده اند.

## ارتباط بین بخش های مختلف پروژه

بخش های مختلف این مخزن به صورت آموزشی و زنجیره ای با یکدیگر مرتبط هستند:

1. `Variables` پایه درک مدل داده و متغیرها را ایجاد می کند.
2. `Encapsulation` بر کنترل داده و طراحی امن کلاس ها بنا می شود.
3. `Extends` و `Implements` ساختارهای شیءگرا را توسعه می دهند و روابط بین کلاس ها و قراردادها را آموزش می دهند.
4. `Overloading` و `Overriding` رفتارهای چندریختی را تکمیل می کنند.
5. `Static` و `Abstract` مفاهیم تکمیلی و طراحی سطح بالاتر را پوشش می دهند.
6. `Exercises` این زنجیره را با تمرین و تثبیت یادگیری کامل می کند.

به همین دلیل، بهترین روش مطالعه این مخزن، دنبال کردن ترتیب سشن ها از جلسه 1 تا جلسه 4 است.

## راهنمای دسترسی به کدهای پروژه

برای مشاهده کامل فایل های سورس پروژه در GitHub، مراحل زیر را دنبال کنید:

1. ابتدا وارد پوشه اصلی مبحث مورد نظر شوید؛ برای مثال `Encapsulation (session 2)`.
2. سپس زیرپوشه پروژه مرتبط را باز کنید؛ برای مثال `Full Example with Encapsulation`.
3. در مرحله بعد، پوشه `src` را باز کنید.
4. تمامی فایل های سورس، کلاس ها و کدهای اصلی توسعه دهنده درون پوشه `src` قرار دارند.

نمونه مسیر صحیح برای دسترسی به سورس:

```text
Encapsulation (session 2)/
+-- Full Example with Encapsulation/
|   +-- src/
|       +-- Main.java
|       +-- Student.java
```

نکته مهم: بازدیدکنندگان GitHub برای مشاهده کامل کدها باید ابتدا وارد پوشه اصلی هر مبحث شوند و سپس پوشه `src` پروژه مربوط به همان مثال را باز کنند. فایل های تنظیمات IntelliJ و فایل های پروژه در کنار سورس قرار دارند، اما فایل های برنامه نویسی اصلی همیشه درون `src` قرار گرفته اند.
